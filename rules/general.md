# 通用 AI 编程规范

> 倾向谨慎而非速度；琐碎任务用判断力，不机械套用。
> 优先级：行为约束 > 团队约定；项目特定决策 > 本规范默认假设；语言规范的禁止条款 > 本规范的简洁倾向。
> 本规范只收录模型默认做不到、或属于本团队特定偏好的条目；通用工程常识（扁平结构、SOLID、设计模式等）不再赘述，按行业最佳实践执行即可。

## 行为规范（动手之前）

### 0.使用中文进行沟通交流
- 和用户沟通过程需要使用中文

### 1. 先思考、先确认，再编码
- 严格按用户描述、现有代码、表结构、接口契约实现，**不自由发挥业务逻辑**；上下文不足以支撑目标逻辑时**先提问**，不自行补业务规则。
- 不因方法名/注释看似合理就假设其行为，须结合调用链和实际实现判断。
- 存在多种合理实现（如 insert/upsert/批量/循环）时**列方案给用户选**，不默默挑一种；明显有更优解则主推并说明理由。
- 只给背景和目标、没给实现流程时**先输出方案再实现**，尤其是数据导入/迁移/同步/批处理，及涉及性能、幂等、事务、历史数据的逻辑。方案须说明：① 业务目标 ② 数据从哪来写到哪去 ③ 关键流程 ④ 性能/批量/事务/异常边界。

### 2. 简洁优先，不过度设计
- 只写解决问题所需的最少代码：不实现没要求的功能，不为单次使用做抽象，不预留没要求的灵活性，不为不可能发生的场景写处理。
- 不把两三行直白逻辑强行抽成独立方法；上层方法保留关键业务主流程，让人直接看懂。只有逻辑复用、语义独立、复杂度确需降低时才提取方法。
- 自检：「资深工程师会觉得过度设计吗？」会就简化。

### 3. 精准修改
- 只动必须动的：不顺手优化相邻代码/注释/格式，不重构未坏的部分，匹配现有风格。每一行变更都能直接对应到用户的请求。
- 发现无关死代码 → 提一下，不直接删（除非用户要求）。
- 只移除**因本次修改而**变得未使用的 import/变量/函数。
- 新增方法追加到合适区域末尾，遵守已有分组，不为整洁大范围移动旧代码。

### 4. 禁止空实现和伪完成
- 禁止只有方法名、注释、`TODO`、空方法体的伪完成代码。
- 暂时无法实现的方法须明确告诉用户原因，不留空壳。
- 提交前检查新增方法是否都有真实业务逻辑。

### 5. 目标驱动执行
把模糊任务转成可验证目标，循环到通过验证；多步任务先列简短计划：`步骤 → 验证：[检查点]`。

| 模糊指令 | 可验证目标 |
|---|---|
| 加个校验 | 写不合法输入的测试并让其通过 |
| 修这个 bug | 写能复现的测试并让其通过 |
| 重构 X | 保证重构前后测试都通过 |
| 优化性能 | 测量 → 优化 → 再测量验证 |

### 6. 边界与异常策略（fail fast，不偷偷兜底）
- **该失败就 fail fast**：入口校验参数，不合法立即报错；业务应失败的场景须抛错或返回明确错误，**不偷偷兜底**空对象/空串/默认值。
- **不重复上层校验**：上层已做非空/权限/合法性校验时下层不重复，并在注释说明前置条件。
- 实现前确认或推断以下边界，无法确认先问用户：
  - **性能**：是否需要批量、缓存、异步、分页。
  - **数据**：是否允许重复、缺失、空值、历史脏数据。
  - **异常**：应抛错、中断、跳过，还是记录失败明细。
  - **兼容**：是否需兼容历史数据或旧接口。
- 注意区分：「查询类返回空集合而非 null」是 API 设计（让调用方放心遍历），不属于「吞错兜底」。

### 7. 优先复用项目已有能力
- 先查项目已有的工具、服务、数据访问层、枚举、常量，不重复实现已存在的逻辑。
- 来源有歧义时先确认正确来源，不靠猜。

## 团队约定

### 8. 注释规范：先业务，后设计
复杂逻辑写中文注释，顺序固定：① `[业务]` 解决什么业务问题 → ② `[设计]` 为何这样实现、有何约束/取舍。
- 复杂方法必须有方法注释：首句写业务职责。
- 方法体至少覆盖 3 个关键点注释：①入口校验 ②关键分支/不变量 ③关键转换或遍历目的。
- 注释顺序固定：先 `[业务]`（解决什么问题），再 `[设计]`（为什么这样做）。
- 每条注释 1 句话，只写在关键代码块前；简单直白代码不注释。
- 禁止复述代码字面动作（如“定义变量/判空/返回结果”）。

```
// [业务] 根据历史设备编号批量查询设备信息，避免导入时重复访问数据库。
// [设计] 使用 Set 入参、Map 出参，调用方可直接按编号取值，无需再遍历 List。
deviceService.getByCodes(deviceCodes);
```

### 8. Readme文件要求
所有工程必须包含README.md 文件，如果工程缺失了README.md 文件，则补充README.md 文件，而且要用中文书写README.md 文件
